Architecture MVC

Ce projet suit le patron Modèle – Vue – Contrôleur.
• Modèle : données pures, aucun import JavaFX ni vue
• Vue : affichage uniquement, pas de logique métier
• Contrôleur : fait le lien, gère les événements et les boucles de jeu

📦 Modèle

  • ModeleEnnemi.javaPosition, vitesse, PV de l'ennemi
  • ModeleTerrain.javaGrille, obstacles, entrée/sortie
  • TypeTuile.javaEnum type de case (sol/mur/coin)
  • ModeleTypeObstacle.javaEnum baril / caillou
  • ModeleDeck.javaListe des cartes du joueur
  • TypeCarte.javaEnum carte avec dégâts et vitesse
  • ModeleProjectile.javaPosition et déplacement d'un projectile
  • BFS.javaAlgorithme de calcul de chemin

⚙ Contrôleur

  • GameController.javaChef d'orchestre – initialise tout
  • ControllerEnnemis.javaDéplace l'ennemi frame par frame
  • ControllerTour.javaAnime la tour, détecte et tire

🖼 Vue

  • VueDeck.javaAffiche les cartes dans les HBox
  • VueEnnemi.javaSprite GIF de l'ennemi
  • VueTerrain.javaRemplit le TilePane avec les tuiles
  • VueObstacles.javaDessine obstacles et portes
  • game-view.fxmlDéclaration XML de l'interface
  • game.cssStyles portée tour, carte sélectionnée
